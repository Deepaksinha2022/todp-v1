# TODP AI Workflow

Founder Responsibilities

- Product vision
- Feature approval
- Testing
- User validation
- Prioritization

AI Responsibilities

- Code generation
- Refactoring
- Testing
- Documentation updates
- Repository structure

Daily Process

1. Update CURRENT_TASK.md
2. Give context files to AI
3. AI implements task
4. Founder tests feature
5. Founder approves/rejects
6. Commit changes
7. Update DAILY_LOG.md

Rule

Every session must produce:
- Code
- Test
- Commit

No architecture redesign unless explicitly approved.
